## ğŸ› ï¸ TROUBLESHOOTING

### Issue: Script Won't Run

**Solution 1**: Enable PowerShell execution
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**Solution 2**: Run script with bypass
```powershell
powershell -ExecutionPolicy Bypass -File .\ORGANIZE_TO_LOCAL.ps1
```

### Issue: Permission Denied

**Solution**: Run PowerShell as Administrator
1. Right-click PowerShell icon
2. Select "Run as Administrator"
3. Navigate to folder and run script

### Issue: Folder Already Exists

**Solution**: The script will overwrite existing folders. If you want to preserve:
```powershell
# Rename existing folder first
Rename-Item "C:\Users\aztec\CLAUDE.MD\Tools\Version_A" -NewName "Version_A_backup_$(Get-Date -Format 'yyyyMMdd')"

# Then run script
.\ORGANIZE_TO_LOCAL.ps1
```

### Issue: Repository Path Not Found

**Solution**: Update `$RepoPath` in script
1. Open `ORGANIZE_TO_LOCAL.ps1` in text editor
2. Change line 13 to your actual repository path:
   ```powershell
   $RepoPath = "C:\Your\Actual\Path\500-AI-Agents-Projects\Tools"
   ```
3. Save and run again
